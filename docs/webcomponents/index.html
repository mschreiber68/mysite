Lately I've been interested in Web Components. My goal is to try and make some Web Components that are:

  • Design system agnostic. I think components that are highly styleable are highly reusable. This probably means limiting the use of the shadow DOM.
  • SEO friendly. Limited use of template HTML.

These components are a work in progress.

Tabs

<mike-tabs> <ul> <li class="active">Tab 1</li> <li>Tab 2</li> <li>Tab 3</li> </ul> <ul> <li class="active">Content 1</li> <li>Content 2</li> <li>Content 3</li> </ul> </mike-tabs>

(Nearly) Unstyled

The tabs component doesn't use the shadow DOM, so it relies on the user to customize all styles, including hiding content that does not have the active class. The example below just highlights whichever tab/content pair would be active. The benefit is that the component can gracefully handle adding and removing tabs.
